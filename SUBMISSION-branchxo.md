### Exact commands

Run from `Untitled/branchxo/`:

```bash
# one-time: link this project to an EAS account & generate projectId
npx eas init

# install/update credentials (interactive)
npx eas credentials -p ios
npx eas credentials -p android

# production builds (in the cloud)
npx eas build -p ios --profile production
npx eas build -p android --profile production

# submit (after the build completes)
npx eas submit -p ios   --latest --profile production
npx eas submit -p android --latest --profile production
```

For iOS submission you must already have:
- Apple Developer Program membership (US$99/year)
- An App Store Connect record for `com.raban.branchxo` (created via App Store
  Connect → My Apps → "+")
- The `ascAppId` from that record copied into `eas.json` →
  `submit.production.ios.ascAppId`

For Android submission you must already have:
- A Google Play Console account (US$25 one-time)
- A Google Play Console app record for `com.raban.branchxo`
- A service-account JSON key with "Release Manager" role, downloaded and
  placed at the path in `eas.json` (`./google-service-account.json`,
  which is in `.gitignore`)

## Per-store final checklist (the literal "Submit for Review" path)

### App Store Connect (iOS)

1. Apple Developer Program enrolled, Team ID known. ⚠️
2. App Store Connect → My Apps → + → New App, fill:
   - Platforms: iOS
   - Name: `branchxo` (or `branchxo: Multiverse Tic-Tac-Toe`)
   - Primary language: English (U.S.)
   - Bundle ID: `com.raban.branchxo` (must match `app.json` — create the
     identifier under Certificates, Identifiers & Profiles first)
   - SKU: `branchxo-1`
3. Fill App Information:
   - Category: Games → Board (Primary), Games → Strategy (Secondary)
   - Content rights: "Does not use third-party content"
   - Age rating: open the Apple questionnaire and answer per
     `STORE-LISTING.md`. Result will be 4+.
4. Fill App Privacy:
   - "Data Collection": **No**
   - (No further fields needed.)
5. Pricing & Availability: Free, available in all territories.
6. Prepare for submission (1.0):
   - Promotional text (170 chars): pull first paragraph from
     `STORE-LISTING.md` full description.
   - Description: full description from `STORE-LISTING.md`.
   - Keywords: 99 chars from `STORE-LISTING.md`.
   - Support URL: GitHub repo URL.
   - Marketing URL: optional.
   - Privacy Policy URL: hosted `PRIVACY.md` URL.
   - Screenshots: upload 6.7" iPhone + 12.9" iPad sets per `STORE-LISTING.md`.
   - App Icon: 1024×1024 (Expo extracts from `assets/icon.png` automatically
     during EAS build).
7. App Review Information:
   - Sign-in not required (no accounts).
   - Notes: "branchxo is an offline 3×3 tic-tac-toe variant with a branching
     timeline. No accounts, no network, no IAP, no data collection."
8. Version Release: "Automatically release this version".
9. Build: pick the cloud-built ipa from `npx eas submit` upload.
10. Click **Submit for Review**.

### Google Play Console (Android)

1. Google Play Console account active, US$25 paid. ⚠️
2. All Apps → Create app:
   - App name: `branchxo`
   - Default language: English (U.S.)
   - App or game: Game
   - Free or paid: Free
   - Accept declarations.
3. Set up your app dashboard tasks:
   - App access: All functionality available without restrictions.
   - Ads: "No, my app does not contain ads"
   - Content rating: complete IARC questionnaire per `STORE-LISTING.md` →
     Everyone.
   - Target audience: 13+ (or all ages — your call; the game is suitable).
   - News app: No.
   - COVID-19 contact tracing: No.
   - Data safety: complete per `STORE-LISTING.md` → "No data collected,
     no data shared."
   - Government app: No.
   - Financial features: None.
   - Health: No.
4. Main store listing:
   - Short description (80 chars).
   - Full description (4000 chars).
   - App icon (Play takes 512×512 — Expo generates from `assets/icon.png`).
   - Feature graphic 1024×500 (📝 must be designed; not generated in this
     pass).
   - Phone screenshots (min 2, up to 8): from `STORE-LISTING.md`.
   - Tablet screenshots (optional but recommended).
5. Production release → Create new release:
   - Upload the `.aab` produced by `npx eas build -p android --profile production`
     (EAS will offer to do this for you via `npx eas submit -p android`).
   - Release name: `1.0.0`
   - Release notes (en-US): from `STORE-LISTING.md` "What's New".
6. Review and start rollout to production → **Send for review**.

## Remaining human steps (the precise minimum to hit "Submit for Review")

In order, end-to-end, assuming this repo as starting point:

1. **Create EAS project.** `cd Untitled/branchxo && npx eas init` — copy
   the generated `projectId` into `app.json` `extra.eas.projectId`.
2. **(iOS)** Enroll in Apple Developer Program; create
   `com.raban.branchxo` App ID in Certificates, Identifiers & Profiles;
   create the App Store Connect record; copy `ascAppId` + `appleTeamId`
   + `appleId` into `eas.json`.
3. **(Android)** Pay Google Play registration; create a Play Console app
   record for `com.raban.branchxo`; create a service-account in Google
   Cloud IAM with "Service Account User" + grant "Release Manager" in
   Play Console; download the JSON key to
   `Untitled/branchxo/google-service-account.json` (gitignored).
4. **Commission/replace placeholder icon + adaptive-icon + splash + Android
   1024×500 feature graphic.** Drop new PNGs into `assets/`.
5. **Host `PRIVACY.md`** at any public HTTPS URL (GitHub Pages works).
   Note the URL.
6. **Capture screenshots.** Boot iOS Simulator + Android emulator;
   `npx expo start` and capture the 5 views listed in `STORE-LISTING.md`
   for each required device class.
7. **Build.**
   ```bash
   cd Untitled/branchxo
   npx eas build -p ios --profile production
   npx eas build -p android --profile production
   ```
8. **Fill listings** in App Store Connect and Google Play Console per
   `STORE-LISTING.md`; upload screenshots.
9. **Submit.**
   ```bash
   npx eas submit -p ios   --latest --profile production
   npx eas submit -p android --latest --profile production
   ```
10. In each store console: click **Submit for Review** (iOS) and
    **Send for review** (Android).

That's the entire path. Steps 1, 5, 7, 9, 10 are the only ones that touch
the codebase; the rest is account configuration and content production.
