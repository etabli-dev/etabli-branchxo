# DECISIONS ‚Äî branchxo

Build-phase notes on design choices and deviations from the spec.

## In-spec choices

### Multiverse representation
Each `Universe` stores ONLY its own divergent moves (`moves`), `parentId`, and
`parentPly`. The full move list at any node is reconstructed by walking up the
parent chain and concatenating `parent.moves.slice(0, parentPly)` recursively
(see `fullMoves`). This keeps every universe immutable and makes "parent
untouched after fork" trivially true: the only data a fork adds is the new
child node.

We also cache `board` and `status` on each universe for O(1) reads. An
invariant test (`assertTreeInvariants`) recomputes them from scratch and
asserts they match ‚Äî guaranteeing the cache cannot drift.

### Strength function
Implemented exactly per spec. For each of the (up to 4) win lines through the
candidate cell, we count my marks `k` and opponent marks `m` on the **other
two cells of the line** (we never count the candidate itself, since it is by
definition empty before placement).
- only mine: `10^k`
- only opp: `10^m * 0.9`
- mixed: `0`
- empty: `1` (centrality)

This yields the required ordering: immediate win (100) > immediate block (90)
> threat-build (10) > centrality (1-4 across the cell's lines). Verified by
unit test.

### AI
- **Easy** uses `Math.random` by default but accepts a deterministic `rng` for
  tests.
- **Medium** picks the max-strength empty cell.
- **Perfect** is full minimax with a `Map<string, MinimaxResult>` cache keyed
  by `boardString|toMove`. We did NOT add symmetry canonicalization ‚Äî the
  cache is small enough already (<5√ó10¬≥ unique reachable positions) and
  adding canonicalization would complicate move-back-translation. Noted in the
  spec as "optional".
- Tie-breaking prefers shorter forced wins and slower forced losses (so the
  AI never "stalls" a winning position).

### Probability modes
- `probPerfect` returns a 0/1 distribution (exact outcome under optimal play).
- `probRandom` runs `R=300` rollouts by default, where both sides play the
  "easy" policy (random + take-win + block-loss). The rollout signature
  accepts both `rng` (for tests) and `shouldCancel` (so the UI can debounce
  and abort).

### Branch UX
We chose a **modal "branch?" confirmation** when the user taps a divergent
cell while scrubbed back, rather than auto-forking. This makes the multiverse
moment explicit and undoable. Forward play (tip-of-timeline) is never
prompted.

### Accessibility
- Cells (`BoardView`) and tree nodes (`MultiverseTree`) carry
  `accessibilityLabel`s describing position / outcome / "X to move".
- Marks are distinguished by **shape + color** (X = cross + blue; O = ring +
  orange) ‚Äî Okabe-Ito accessible pair, so total-color-blind users still
  perceive the difference.
- Outcome colors on tree nodes also use distinct hues + shape changes (active
  node has a halo ring).

### Persistence
We persist the entire tree (`MultiverseTree`) plus all settings under
`branchxo:v1`. On hydrate we run `assertTreeInvariants` defensively ‚Äî if the
stored tree fails the invariant check (e.g. corrupted), we silently reset to a
single root rather than crash.

## Deviations / deferrals

1. **Skia tree pan/zoom** is implemented with nested ScrollViews rather than a
   gesture-driven matrix transform. This is sufficient for trees up to ~50+
   nodes and ships immediately. A true pinch-zoom with `react-native-gesture-handler`
   + Reanimated worklets is on the polish/refine list (audit phase will flag
   if needed).
2. **Skia "O" mark** is drawn with a stroked `Circle` rather than via a Path ‚Äî
   semantically equivalent, code is simpler.
3. **Symmetry canonicalization** in the minimax cache is intentionally
   omitted (see above). The cache is fast enough.
4. **Reanimated** is installed and the Babel plugin is configured, but the
   build pass does not yet use reanimated worklets for tile animations. This
   is in scope for the refine/polish pass.
5. **App icons / splash** are placeholder PNGs generated by a Node script in
   `assets/`. Matching SVG sources are in `assets/*.svg`. Final art will be
   produced for the submission pass.
6. **Web preview** works for layout but the touch hit-tests on cells are sized
   for native; not tuned for desktop pointer.
